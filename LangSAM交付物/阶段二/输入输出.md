# LangSAM 输入输出整理

## 1. 核心模型接口

核心接口位于 `lang_sam/lang_sam.py` 中，主要调用方式如下：

```python
from PIL import Image
from lang_sam import LangSAM

model = LangSAM()
image_pil = Image.open("./assets/car.jpeg").convert("RGB")
text_prompt = "wheel."

results = model.predict(
    images_pil=[image_pil],
    texts_prompt=[text_prompt],
    box_threshold=0.3,
    text_threshold=0.25,
)
```

函数原型可整理为：

```python
predict(
    images_pil: list[Image.Image],
    texts_prompt: list[str],
    box_threshold: float = 0.3,
    text_threshold: float = 0.25,
) -> list[dict]
```

### 1.1 输入

| 输入项 | 类型 | 说明 | 示例 |
|---|---|---|---|
| `images_pil` | `list[PIL.Image.Image]` | 输入图片列表，要求为 RGB 格式 | `[Image.open("./assets/car.jpeg").convert("RGB")]` |
| `texts_prompt` | `list[str]` | 文本提示词列表，与图片一一对应 | `["wheel."]` |
| `box_threshold` | `float` | 目标检测框阈值 | `0.3` |
| `text_threshold` | `float` | 文本匹配阈值 | `0.25` |

说明：

- `images_pil` 不是图片路径，而是已经读入内存的 `PIL.Image` 对象。
- `texts_prompt` 必须使用列表形式传入。
- 当只处理一张图片时，也需要写成单元素列表。

### 1.2 输出

`predict()` 返回值类型为 `list[dict]`，每张图片对应一个结果字典。

单张图片的结果通常通过以下方式获取：

```python
result = results[0]
```

单个结果字典的主要输出字段如下：

| 输出项 | 类型 | 说明 |
|---|---|---|
| `boxes` | `np.ndarray` | 检测到的目标框坐标 |
| `scores` | `np.ndarray` | 检测框置信度分数 |
| `labels` | `list[str]` | 检测到的文本标签 |
| `masks` | `np.ndarray` 或空列表 | 分割掩码结果 |
| `mask_scores` | `np.ndarray` 或空列表 | 掩码质量分数 |

结果结构示意如下：

```python
[
    {
        "boxes": ...,
        "scores": ...,
        "labels": ...,
        "masks": ...,
        "mask_scores": ...
    }
]
```

## 2. 终端封装接口

为了不依赖 Gradio 前端，项目中新增了 `cli_predict.py`，用于在终端中直接完成“输入图片路径和文本 prompt -> 模型推理 -> 输出结果文件”的完整流程。

调用方式如下：

```powershell
python .\cli_predict.py --image <图片路径> --prompt <文本提示词> --output-dir <输出目录>
```

例如：

```powershell
python .\cli_predict.py --image .\assets\car.jpeg --prompt "wheel." --output-dir .\outputs\car_result
```

### 2.1 输入

| 输入项 | 类型 | 说明 | 示例 |
|---|---|---|---|
| `--image` | `str` | 输入图片路径 | `./assets/car.jpeg` |
| `--prompt` | `str` | 文本提示词 | `"wheel."` |
| `--output-dir` | `str` | 输出结果保存目录 | `./outputs/car_result` |
| `--sam-type` | `str` | SAM 模型类型 | `sam2.1_hiera_small` |
| `--box-threshold` | `float` | 检测框阈值 | `0.3` |
| `--text-threshold` | `float` | 文本匹配阈值 | `0.25` |

### 2.2 输出

终端脚本会在输出目录下生成以下文件：

| 输出项 | 类型 | 说明 | 用途 |
|---|---|---|---|
| `mask.png` | 图片文件 | 分割掩码图 | 用于目标区域提取或透明 PNG 生成 |
| `result.png` | 图片文件 | 分割可视化结果图 | 用于前端展示 |
| `result.json` | JSON 文件 | 保存结构化检测结果 | 供后端读取并返回给前端 |

其中，`result.json` 包含以下主要字段：

| 字段 | 说明 |
|---|---|
| `image_path` | 输入图片路径 |
| `text_prompt` | 输入文本 prompt |
| `status` | 运行状态，通常为 `success` 或 `no_target` |
| `mask` | mask 文件路径 |
| `result_image` | 结果图路径 |
| `boxes` | 检测框信息 |
| `scores` | 检测分数 |
| `labels` | 标签信息 |
| `mask_scores` | mask 分数 |

## 3. 简要说明

从项目现有结构来看，LangSAM 的核心模型接口本身并不直接接收“图片路径”，而是接收已经加载好的图片对象 `PIL.Image`。如果后端希望直接通过路径调用模型，则需要像 `cli_predict.py` 一样，先完成图片读取，再将图片对象传入 `model.predict()`。

因此，可以将整个输入输出关系概括为：

1. 终端或后端接收 `image_path` 和 `text_prompt`。
2. 程序将图片路径读取为 `PIL.Image` 对象。
3. 调用 `LangSAM.predict()` 完成文本目标检测与分割。
4. 输出 `masks`、`boxes`、`scores` 等结构化结果。
5. 将结果进一步保存为 `mask.png`、`result.png` 和 `result.json`，供后端读取后返回前端。
