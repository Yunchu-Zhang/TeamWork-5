# 接入问题记录（LangSAM -> `/segment/lang`）

本文件用于记录后端接入 LangSAM 时常见报错原因与处理方式。

## 1. 没进 conda 环境导致缺依赖

现象：

- `ModuleNotFoundError: No module named 'gradio'/'litserve'/'transformers'...`
- `环境测试.py` 提示多个依赖未安装

原因：

- 没有进入 `sam2_env`，而是在系统默认 Python 环境运行

解决：

- `conda activate sam2_env`
- 在项目目录执行 `python 环境测试.py` 确认依赖齐全

## 2. prompt 写中文或太抽象，检测不到目标

现象：

- 返回结果 `labels` 为空，`masks` 为空，或服务端日志提示 `No masks detected`

原因：

- prompt 用中文、句子太长、或词过于抽象（例如 `thing`）

解决：

- 仅使用英文名词：`cup.` / `person.` / `chair.`
- 必要时调低阈值（例如 `box_threshold=0.25`、`text_threshold=0.2`）再测试

## 3. SAM2 权重找不到/路径不对

现象：

- 启动时加载权重失败，报 `FileNotFoundError` 或提示 checkpoint 不可用

原因：

- 权重文件缺失，或不在项目约定目录

解决：

- 权重目录：`A:\_Sofaware\项目\lang-segment-anything-main\torch\hub\checkpoints`
- 确认存在：`sam2.1_hiera_small.pt`

## 4. 第一次运行需要下载 GroundingDINO 权重（离线会失败）

现象：

- 第一次预测时卡住或报 HuggingFace 下载/网络相关错误

原因：

- GroundingDINO 默认从 Hugging Face Hub 拉取

解决：

- 确保能联网完成首次下载，或把模型下载到本地目录并在初始化时指定：
  - `gdino_model_ckpt_path`
  - `gdino_processor_ckpt_path`

## 5. GPU/显存问题导致推理慢或 OOM

现象：

- 推理很慢，或报显存不足（CUDA OOM）

原因：

- 图片太大、并发太高、或选择了更大的 SAM2 模型

解决：

- 先用 `sam2.1_hiera_small`
- 限制图片最大边尺寸（例如后端先 resize）
- 控制并发/队列化请求
