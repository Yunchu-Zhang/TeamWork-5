## 2. 各文档用途

### 2.1 `LangSAM调用补充说明.md`

这份文档主要面向后端接入，内容重点是：

- LangSAM 推荐调用方式
- `/segment/lang` 接口接入思路
- `LangSAM.predict(...)` 的输入参数和返回结构
- mask 导出建议
- 标注图导出方式
- 是否采用 HTTP `/predict` 服务方式

适合在准备实际接接口时优先阅读。

### 2.2 `输入输出.md`

这份文档主要说明 LangSAM 的模型接口输入输出格式，包括：

- 核心模型接口输入输出
- 终端封装脚本输入输出
- 结构化结果字段说明

适合在前后端联调前统一字段认知时使用。

### 2.3 `问题记录.md`

这份文档记录了接口整理和调试过程中遇到的典型问题，例如：

- 为什么一开始会误以为只能前端调用
- 为什么 `image_path` 不能直接等同于模型函数输入
- 为什么需要补充终端调用脚本
- 为什么结果需要额外落盘为文件

适合在遇到理解偏差、调用不通、结果结构不一致时查阅。

### 2.4 `稳定prompt示例.md`

这份文档主要给出较稳定的英文 prompt 示例，便于快速测试和验收，例如：

- `person.`
- `car.`
- `wheel.`
- `cup.`

适合前后端在联调或演示时直接复用，减少因为 prompt 不稳定导致的误判。

### 2.5 `文本分割样例/README.md`

这份文档和同目录图片主要用于展示已有输入输出样例，方便进行：

- 效果对照
- 联调结果核验
- 回归测试参考

如果后续 `/segment/lang` 接口联调完成，也可以继续往这个目录补充真实接口输出样例。

## 4. 前后端各自重点关注内容

### 4.1 后端重点关注

- 重点看 `LangSAM.predict(...)` 的输入输出结构。
- 明确图片在模型层不是路径，而是 `PIL.Image` 对象。
- 明确返回结果里最关键的是 `masks`、`boxes`、`scores`、`labels`。
- 如果要做 PNG 抠图，优先关注 mask 的保存逻辑。
- 如果要做接口落地，优先参考 `LangSAM调用补充说明.md` 中 `/segment/lang` 的伪代码。

### 4.2 前端重点关注

- 重点看接口需要传入的 `image` 和 `text_prompt`。
- 明确 prompt 建议使用英文名词，必要时由前端提供目标选择项。
- 前端展示建议优先使用结果图 `result.png` 或后端返回的可视化图片。
- 如果需要做“分割成功/未命中目标”的提示，应结合 `status` 字段或后端返回状态判断。
- 联调测试时优先用 `稳定prompt示例.md` 中的词，减少测试波动。
