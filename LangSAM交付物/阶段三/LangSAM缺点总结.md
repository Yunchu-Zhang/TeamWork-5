# LangSAM 缺点总结

## 1. 复杂背景下不稳定

当背景里有很多相似纹理、颜色或干扰物时，GroundingDINO 可能定位不准，导致后面的 SAM2 mask 也跟着不准。

常见表现：

- 识别到错误目标
- mask 边缘不完整
- 框住目标旁边的背景

## 2. 多个同类物体时可能选错

如果一张图中有很多同类物体，例如多把椅子、多本书、多个人，prompt 只写 `chair.` 或 `person.` 时，模型可能会识别多个目标，也可能只选中其中一部分。

这类情况对后端和前端都需要额外处理，例如让用户选择其中一个结果，或者返回多个 mask。

## 3. 小目标容易漏检

目标太小、太远或被遮挡时，文本检测阶段可能找不到目标。即使找到了，分割出的 mask 也可能比较粗糙。

例如：

- 远处的小杯子
- 桌面角落的小手机
- 背景中很小的书本

## 4. prompt 依赖英文表达

LangSAM 更适合英文目标词。中文 prompt、过长描述或抽象词会降低稳定性。

不建议使用：

| Prompt | 问题 |
| --- | --- |
| `东西` | 太抽象 |
| `thing.` | 太宽泛 |
| `桌子上那个比较亮的物体` | 描述太长且中文 |

建议使用简单英文名词，例如 `cup.`、`book.`、`person.`。

## 5. 阈值会影响结果

`box_threshold` 和 `text_threshold` 设置过高时，可能检测不到目标；设置过低时，可能引入错误目标。
