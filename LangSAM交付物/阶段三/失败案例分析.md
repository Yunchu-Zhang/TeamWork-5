# 失败案例分析

## 1. 未进入正确环境导致失败

对应截图：

```text
LangSAM交付物\阶段一\案例截图\失败案例1_当前环境依赖缺失无法启动.png
```

失败原因：

- 直接使用系统默认 Python 运行，而不是进入 `sam2_env`。
- 默认环境缺少 `gradio`、`litserve`、`transformers`、`sam-2` 等依赖。

解决方式：

```text
cd /d A:\_Sofaware\项目\lang-segment-anything-main
conda activate sam2_env
python 环境测试.py
python app.py
```

## 2. 未输入图片导致失败

对应截图：

```text
LangSAM交付物\阶段一\案例截图\失败案例2_未输入图片.png
```

失败原因：

- LangSAM 必须同时接收图片和文本 prompt。
- 只输入 prompt，不上传图片，模型没有可分割对象。

解决方式：

- 前端调用 `/segment/lang` 时必须上传图片文件。
- 后端需要校验 `image` 字段，缺失时返回明确错误信息。

## 3. 目标太小或背景复杂导致失败

可能失败场景：

- 远处的小杯子、小手机、小书本。
- 背景颜色与目标颜色接近。
- 目标被遮挡，只露出一部分。
- 图中有大量相似纹理，例如书架、教室、拥挤街景。

失败原因：

- GroundingDINO 可能找不到目标或检测框不准。
- SAM2 依赖前一步的检测框，如果框不准，mask 也会跟着不准。

解决建议：

- 换更明确的 prompt，例如 `cup.` 不要写 `thing.`。
- 适当降低 `box_threshold` 或 `text_threshold`。
- 对图片先裁剪，让目标在画面中更大。

## 4. 多个同类物体导致结果不稳定

可能失败场景：

- 一张图中有很多椅子，prompt 为 `chair.`。
- 一张图中有很多人，prompt 为 `person.`。
- 桌面上有多本书，prompt 为 `book.`。

失败原因：

- LangSAM 不知道用户具体想要哪一个同类目标。
- 可能返回多个 mask，也可能只返回其中几个。

解决建议：

- 后端保留多个 mask，前端让用户选择。
- 或者改用 SAM2 点选分割，让用户点击具体目标。

## 5. 阈值设置不合适导致失败

相关截图：

```text
LangSAM交付物\阶段一\案例截图\低文本置信度.png
LangSAM交付物\阶段一\案例截图\高文本置信度.png
LangSAM交付物\阶段一\案例截图\低边界框置信度.png
LangSAM交付物\阶段一\案例截图\高边界框置信.png
```

失败原因：

- 阈值太高：模型过于严格，可能检测不到目标。
- 阈值太低：模型过于宽松，可能把背景或错误目标也识别出来。

建议默认值：

```text
box_threshold = 0.3
text_threshold = 0.25
```
