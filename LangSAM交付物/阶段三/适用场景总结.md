# LangSAM 适用场景总结

## 1. 最适合的场景

LangSAM 适合“单个目标明显、目标名称明确”的图片。

典型场景：

| 场景 | 示例 prompt | 适合原因 |
| --- | --- | --- |
| 人物分割 | `person.` | 常见类别，识别稳定 |
| 车辆分割 | `car.` | 目标整体清晰 |
| 轮胎分割 | `wheel.` | 形状明显，边界清楚 |
| 杯子分割 | `cup.` | 名称明确，适合文本输入 |
| 书本分割 | `book.` | 适合桌面或简单背景 |
| 椅子分割 | `chair.` | 单个椅子效果较好 |

## 2. 适合项目中的文本分割入口

在本项目中，LangSAM 适合作为 `/segment/lang` 的算法支持。

推荐流程：

```text
前端上传图片 -> 用户输入英文 prompt -> 后端调用 LangSAM -> 保存 mask -> 生成透明 PNG -> 前端展示/下载
```

这种流程适合用户知道目标名称，但不想手动点选目标的情况。

## 3. 不太适合的场景

LangSAM 不太适合以下情况：

- 目标特别小。
- 背景特别复杂。
- 同类物体数量很多。
- 用户想精确选择某一个相同类别目标。
- prompt 无法用简单英文名词表达。

这些场景更适合使用 SAM2 点选分割，让用户通过点击告诉模型具体目标位置。

## 4. 推荐使用策略

项目里可以把 LangSAM 和 SAM2 点选分割配合使用：

| 用户需求 | 推荐方法 |
| --- | --- |
| 只知道目标名称 | LangSAM 文本分割 |
| 想快速分割明显目标 | LangSAM 文本分割 |
| 要选多个不同类别目标 | LangSAM 多 prompt |
| 要精确选择某一个目标 | SAM2 点选分割 |
| 文本识别错目标 | 切换到 SAM2 点选分割 |

## 5. 阶段三结论

LangSAM 的优势是“输入简单、接入清楚、适合明确目标”。它非常适合作为项目中的文本分割入口。

但它不是所有图片都稳定。复杂背景、小目标、多个同类物体时，建议保留 SAM2 点选分割作为补充方案。
