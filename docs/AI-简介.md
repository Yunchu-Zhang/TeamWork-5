
# 项目简介

本项目拟设计并实现一个**基于 SAM2 与 LangSAM 的场景图像目标分割及 PNG 素材生成系统**。系统面向日常场景图片中的具体物品提取需求，用户可以上传一张包含多个物体的场景图，例如桌面、教室、宿舍、街景或室内环境图片，然后通过**点选目标**或**输入目标名称**的方式指定需要提取的物品。后端调用相应的图像分割算法生成目标区域 mask，并进一步将目标物体从原图中抠出，生成透明背景的 PNG 素材图，最终在前端进行展示和下载。

本项目的核心思想是将图像分割模型与前后端系统结合起来，不仅验证模型在实际图片中的分割效果，也实现一个具有实际应用价值的图像素材提取工具。相比单纯的图像分割实验，本项目更加关注“用户如何选择目标”“后端如何调用算法”“分割结果如何转化为可使用的 PNG 素材”以及“不同算法方式在不同场景中的适用性”。

# 一、项目目标

本项目主要目标包括以下几个方面：

1. **实现图片上传与预览功能**  
    用户可以在前端上传一张场景图片，并在页面中查看原图。
    
2. **实现两种目标选择方式**  
    第一种是基于 SAM2 的点选分割，用户在图片中点击目标物体，系统根据点击坐标完成分割。  
    第二种是基于 LangSAM 的文本分割，用户输入目标名称，例如 `cup`、`book`、`chair`，系统自动识别并分割对应物体。
    
3. **实现目标区域 mask 生成**  
    后端根据用户选择方式调用对应算法，生成目标物体的 mask 图像。
    
4. **实现透明背景 PNG 素材生成**  
    系统根据 mask 将目标物体从原图中提取出来，并生成透明背景的 PNG 素材图。
    
5. **实现结果展示与下载功能**  
    前端展示原图、mask 图、PNG 结果图，并提供下载功能。
    
6. **完成双算法效果对比分析**  
    对比 SAM2 点选分割和 LangSAM 文本分割在不同场景下的效果，总结两种方式的优缺点和适用范围。

# 二、系统具体流程

本项目的整体流程可以概括为：

> 用户上传场景图片  
> → 选择目标提取方式  
> → 后端调用对应算法  
> → 生成 mask  
> → 转换为透明 PNG  
> → 前端展示与下载  
> → 测试与效果分析

具体流程如下。

## 1. 图片上传

用户首先在前端页面上传一张场景图片。图片可以是桌面、教室、宿舍、街景、房间等真实场景图。上传后，前端会显示原始图片，方便用户选择需要提取的目标物体。

## 2. 目标选择

系统提供两种目标选择方式。

第一种是**点选分割模式**。用户直接在图片上点击想要提取的物体，例如点击杯子中心、椅子区域或电脑屏幕区域。前端记录点击坐标，并将图片和坐标一起发送到后端。

第二种是**文本分割模式**。用户在输入框中输入目标名称，例如 `cup`、`book`、`chair`、`person` 等。系统将文本提示词发送给后端，由 LangSAM 根据文本描述寻找并分割目标物体。

## 3. 后端算法处理

后端根据用户选择的模式调用不同算法。

如果用户使用点选模式，后端调用 SAM2 点选分割模块，根据图片和点击坐标生成目标 mask。

如果用户使用文本模式，后端调用 LangSAM 文本分割模块，根据图片和文本提示词自动定位目标，并生成对应 mask。

## 4. PNG 素材生成

算法生成 mask 后，系统进一步进行图像后处理。后端根据 mask 保留目标物体区域，将背景区域设置为透明，最终生成透明背景 PNG 图像。

这一部分是本项目从“图像分割实验”转向“素材生成应用”的关键步骤。

## 5. 前端结果展示

前端接收到后端返回的结果后，会展示三类内容：

- 原始图片；
    
- mask 分割结果；
    
- 透明背景 PNG 素材图。
    

用户可以直观看到目标物体是否被正确提取，并可以下载生成的 PNG 文件。

## 6. 结果测试与算法对比

最后，使用不同场景图片对系统进行测试，记录 SAM2 点选分割和 LangSAM 文本分割的效果。测试内容包括分割是否成功、边缘是否准确、PNG 是否生成正常、复杂背景下是否容易失败等。通过测试结果分析两种算法的适用范围。

# 三、系统功能模块

本项目可以分为五个主要模块。

|模块|主要功能|
|---|---|
|前端交互模块|图片上传、图片预览、点选目标、文本输入、结果展示、PNG 下载|
|后端服务模块|接收图片和参数，调用算法，返回结果路径|
|SAM2 点选分割模块|根据用户点击坐标生成目标物体 mask|
|LangSAM 文本分割模块|根据用户输入的目标名称自动识别并分割目标|
|PNG 输出模块|根据 mask 提取目标物体，生成透明背景 PNG 素材|

# 四、项目适用范围

本项目适合用于需要从场景图片中提取单个或少量目标物体的场景，尤其适合以下应用：

## 1. PPT 或课程汇报素材提取

用户可以从图片中提取杯子、书本、电脑、人物、建筑等物体，生成透明 PNG 后直接用于 PPT、海报或课程展示。

## 2. 设计素材快速生成

对于简单设计任务，可以快速从真实图片中提取目标物体，作为海报、宣传图、网页素材或贴纸素材。

## 3. 图像编辑辅助工具

本系统可以作为简化版抠图工具，帮助用户快速获得目标物体的透明背景图。

## 4. 计算机视觉课程实验

项目包含前端、后端、图像分割算法、文件处理和结果评价，适合作为计算机视觉或人工智能方向的综合实验项目。

## 5. 算法对比实验

通过 SAM2 点选分割和 LangSAM 文本分割的对比，可以观察不同输入方式对图像分割效果的影响。

# 五、适用场景与限制

## 适合的场景

本项目更适合以下情况：

- 图片中目标物体比较清晰；
    
- 目标与背景边界较明显；
    
- 用户知道自己想提取哪个物体；
    
- 目标物体不是特别小；
    
- 图片中同类物体数量不太多；
    
- 文本提示词比较明确，例如 `cup`、`book`、`chair`。
    

例如，桌面上的杯子、书本、键盘，教室中的椅子、桌子，街景中的车辆、路牌，都比较适合进行分割和 PNG 提取。

## 不太适合的场景

本项目在以下情况下可能效果不稳定：

- 图片背景非常复杂；
    
- 目标物体太小或被遮挡；
    
- 多个同类物体同时出现；
    
- 目标边缘模糊；
    
- 文本 prompt 不准确；
    
- LangSAM 无法准确理解用户输入的目标名称；
    
- SAM2 点选位置不准确，导致分割到错误区域。
    

例如，一张教室图片中有很多椅子，如果用户只输入 `chair`，LangSAM 可能无法准确判断用户想提取哪一把椅子；而 SAM2 点选模式在这种情况下可能更稳定，因为用户可以直接点击具体目标。

# 六、两种算法方式的特点

|对比维度|SAM2 点选分割|LangSAM 文本分割|
|---|---|---|
|输入方式|用户点击目标位置|用户输入目标名称|
|操作特点|需要手动点选，但目标更明确|更智能，但依赖文本识别效果|
|适用场景|多目标、复杂背景、需要精确指定目标|单目标明显、目标名称清楚的图片|
|优点|用户控制性强，较稳定|操作简单，不需要手动点击|
|缺点|需要用户参与|可能识别错目标或漏检|
|项目定位|主功能算法|辅助算法和对比算法|
