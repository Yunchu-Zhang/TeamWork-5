# 项目简介

## 一段话项目简介：
  本项目围绕真实校园场景，设计并实现了一个基于 SAM2 与 LangSAM 的交互式图像目标分割及 PNG 素材生成系统，同时对于两种图像分割算法进行效果测试与对比。用户可以上传教室、宿舍、图书馆、食堂、桌面学习场景等校园图片，并通过“点选目标”或“输入目标名称”两种方式选择需要提取的物体；系统后端根据用户选择调用 SAM2 点选分割或 LangSAM 文本分割方法生成目标区域 mask，再将目标物体从原图中提取出来，生成透明背景 PNG 素材，并在前端完成原图、mask 图和 PNG 结果图的展示与下载。本项目的重点在于构建一个完整的交互式图像素材提取系统，并通过多类校园场景测试，对比点选式目标选择和文本式目标选择在目标识别准确性、边缘完整度、背景残留、多目标干扰和操作便利性等方面的实际效果，从而分析两种方式的适用范围、优势和局限性。

## 一、项目背景

在课程汇报、PPT 制作、海报设计、图像编辑和学习资料整理等场景中，用户经常需要从真实图片中提取某个具体物体，例如从教室图片中提取椅子、从桌面图片中提取书本或水杯、从宿舍图片中提取电脑和台灯等。传统抠图工具通常需要较多手动操作，而普通图像分割实验又往往停留在算法输出层面，缺少面向真实使用场景的交互流程设计。

基于这一问题，本项目围绕真实校园场景，设计并实现了一个场景图像目标分割与 PNG 素材生成系统。项目重点关注如何将已有图像分割能力转化为一个可交互、可使用、可对比分析的完整系统。用户可以上传校园场景图片，通过点选目标或输入目标名称两种方式选择需要提取的物体，系统自动生成目标 mask，并进一步输出透明背景 PNG 素材。



---

## 二、项目定位

本项目可以定位为一个面向校园场景的交互式图像素材提取工具，同时也是一个用于比较不同目标选择方式实际效果的实验平台。

系统主要面向以下问题：

1. 在真实校园场景图片中，用户如何方便地指定想要提取的目标？
2. 点选式目标选择和文本式目标选择在实际使用中各有什么优势和局限？
3. 分割模型生成的 mask 如何进一步转化为可直接使用的透明背景 PNG 素材？
4. 在教室、宿舍、图书馆、食堂、桌面学习场景等不同校园图片中，两种方式的分割效果是否稳定？

因此，本项目是将图像分割、前后端交互、PNG 后处理和多场景测试结合起来，形成从“用户选择目标”到“生成可用素材”的完整流程。

## 三、系统功能简介

本项目设计并实现了一个基于 SAM2 与 LangSAM 的校园场景图像目标分割及 PNG 素材生成系统。用户可以在前端上传真实校园场景图片，并通过两种方式选择目标物体：

第一种是 **SAM2 点选分割方式**。用户直接在图片上点击想要提取的物体，系统根据点击坐标生成对应的目标区域 mask。这种方式更适合多个同类物体同时出现、需要精确指定某一个目标的情况。

第二种是 **LangSAM 文本分割方式**。用户输入目标名称，例如 `cup`、`book`、`chair`、`laptop` 等，系统根据文本提示自动识别并分割图像中的目标物体。这种方式操作更自然，适合目标名称明确、图片中目标较明显的情况。

后端在获得目标 mask 后，会进一步进行图像后处理，将目标物体从原图中提取出来，并生成透明背景 PNG 素材。前端最终展示原图、mask 图和 PNG 结果图，并提供结果下载功能。

## 四、系统整体流程

本项目的整体流程如下：

> 上传校园场景图片  
> → 选择目标提取方式  
> → 点选目标或输入目标名称  
> → 后端调用对应分割算法  
> → 生成目标区域 mask  
> → 根据 mask 生成透明背景 PNG  
> → 前端展示原图、mask 图和 PNG 结果图  
> → 用户下载 PNG 素材  
> → 记录测试结果并进行算法对比分析

通过这一流程，系统实现了从图片输入、交互选择、算法分割、结果生成到素材下载的完整闭环。

## 五、技术路线

本项目采用前后端分离的基本思路。

前端部分主要负责用户交互，包括图片上传、图片预览、目标点选、文本输入、算法方式选择、结果展示和 PNG 下载。页面重点体现“交互式媒体”的特点，使用户能够直观看到图片、选择目标并查看分割结果。

后端部分主要负责图片文件处理、算法调用和结果生成。系统接收前端上传的图片以及用户选择参数，根据不同模式调用 SAM2 或 LangSAM 完成目标分割，生成 mask 图像，并进一步将目标区域从原图中提取为透明背景 PNG 文件。后端统一返回原图路径、mask 路径和 PNG 结果路径，方便前端展示和下载。

算法部分主要包括两类目标分割方式：

| 算法方式 | 输入内容 | 输出内容 | 主要作用 |
|---|---|---|---|
| SAM2 点选分割 | 图片 + 点击坐标 | 目标 mask | 根据用户点击位置精确分割目标 |
| LangSAM 文本分割 | 图片 + 文本 prompt | 目标 mask | 根据目标名称自动识别并分割目标 |

PNG 生成部分负责根据 mask 保留目标区域，并将背景区域设置为透明，最终生成可用于 PPT、海报或图像编辑的透明背景素材。

## 六、多校园场景测试设计

为了避免项目只停留在单张图片演示层面，本项目将围绕多类校园真实场景进行测试。测试场景包括但不限于：

| 场景类型 | 示例目标 |
|---|---|
| 教室场景 | 桌子、椅子、书本、黑板、投影设备 |
| 宿舍场景 | 水杯、台灯、电脑、书包、床上用品 |
| 图书馆场景 | 书本、电脑、座椅、书包 |
| 食堂场景 | 餐盘、水杯、桌椅 |
| 桌面学习场景 | 笔记本、鼠标、键盘、水杯、书 |
| 校园户外场景 | 路牌、人物、车辆、建筑局部 |

测试过程中会分别使用 SAM2 点选分割和 LangSAM 文本分割处理同一批图片，并记录两种方式在不同场景下的表现。

## 七、双算法效果对比思路

本项目的重点之一是比较两种目标选择方式的实际效果。对比不是只看算法是否能运行，而是结合真实校园场景，从用户操作和结果质量两个角度进行分析。

主要对比维度包括：

| 对比维度 | 说明 |
|---|---|
| 目标识别准确性 | 是否分割到了用户真正想要的物体 |
| 边缘完整度 | 目标边缘是否完整，是否有明显缺失 |
| 背景残留程度 | 生成 PNG 后是否仍残留背景区域 |
| 多目标干扰 | 多个同类物体出现时是否容易选错 |
| 复杂背景适应性 | 背景较杂乱时结果是否稳定 |
| 操作便利性 | 点选和文本输入哪种方式更方便 |
| 适用场景 | 不同方式更适合哪些校园图片 |

预期来看，SAM2 点选分割在需要精确指定目标时更稳定，尤其适合多个同类物体同时出现的场景；LangSAM 文本分割交互更自然，但在复杂背景、目标较小或同类物体较多时可能出现识别错误或漏检。因此，本项目将通过测试表格、成功案例、失败案例和结果截图对两种方式进行综合分析。

## 八、项目特色

本项目的特色主要体现在以下几个方面：

1. **面向真实校园场景**  
   项目不只使用简单样例图片，而是围绕教室、宿舍、图书馆、食堂等校园场景进行测试，使系统更贴近学生日常使用需求。

2. **强调完整交互流程**  
   系统覆盖图片上传、目标选择、算法分割、mask 展示、PNG 生成和结果下载，形成完整的交互闭环。

3. **比较两种目标选择方式**  
   项目将 SAM2 点选分割和 LangSAM 文本分割放在同一系统中进行对比，分析“点击选择”和“文本选择”在实际使用中的差异。

4. **从分割结果转向素材生成**  
   项目不仅输出 mask，还进一步生成透明背景 PNG，使结果可以直接用于 PPT、海报和图像编辑。

5. **结合测试数据进行分析**  
   项目通过多场景测试表、成功案例和失败案例总结两种方式的适用范围，而不是只做单次演示。
